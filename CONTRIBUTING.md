# Contributing

Contributions are welcome when they preserve the distinction between proposed mechanisms, synthetic smoke evidence and qualified experimental results.

## Pull requests

1. State the scientific question or defect being addressed.
2. Add or update tests for metric and model changes.
3. Keep oracle context factors out of deployable inference paths.
4. Label synthetic, development and qualified artifacts explicitly.
5. Update the claim manifest when a change affects an admissible claim.
6. Do not remove failed or negative results merely because they are unfavorable.

## Result contributions

A result pull request should include the configuration, run manifest, per-seed CSV files, aggregation code and a note identifying which gate in `docs/CLAIMS_AND_GATES.md` it addresses.
